    2Early Errors

    2.1Object Initializer

    ObjectLiteral:{PropertyDefinitionList} ObjectLiteral:{PropertyDefinitionList,}
    • It is a Syntax Error if HiddenNameList of PropertyDefinitionList contains duplicate entries.

    2.2Method Definitions

    PropertyDefinition:HiddenMethod
    • It is a Syntax Error if HiddenMethodName of HiddenMethod is empty.

    2.3Class Definitions

    ClassBody:ClassElementList
    • It is a Syntax Error if PrototypePropertyNameList of ClassElementList contains more than one occurrence of "constructor".
    • It is a Syntax Error if HiddenNameList of ClassElementList contains duplicate entries.
    • It is a Syntax Error if HiddenNameList of ClassElementList contains "constructor".
    • It is a Syntax Error if HiddenNameList of ClassElementList contains "prototype".
    • It is a Syntax Error if MultipleClassInitializers of ClassElementList is true.

    2.4The delete operator

    Editor's Note

    Hidden member expressions may not appear as the operand of a delete operator.

     UnaryExpression:deleteUnaryExpression
    • It is a Syntax Error if the UnaryExpression is contained in strict mode code and the derived UnaryExpression is PrimaryExpression:IdentifierReference .
    • It is a Syntax Error if the derived UnaryExpression is MemberExpression:MemberExpression->IdentifierName or CallExpression:CallExpression->IdentifierName .
    • It is a Syntax Error if the derived UnaryExpression is
      PrimaryExpression:CoverParenthesizedExpressionAndArrowParameterList
      and CoverParenthesizedExpressionAndArrowParameterList ultimately derives a phrase that, if used in place of UnaryExpression, would produce a Syntax Error according to these rules. This rule is recursively applied.
    Note

    The last rule means that expressions such as delete (((foo))) produce early errors because of recursive application of the first rule.

    3Static Semantics

    3.1PropName

    Editor's Note

    PropName returns empty for hidden method definitions

     PropertyDefinition:HiddenMethod ClassElement:HiddenMethod
    1. Return empty.

    3.2HiddenMethodName

    Editor's Note

    HiddenMethodName of a HiddenMethod is empty unless it derives a method with an IdentifierName property name.

     HiddenMethod:hiddenMethodDefinition
    1. Return HiddenMethodName of MethodDefinition.
    MethodDefinition:PropertyName(UniqueFormalParameters){FunctionBody} MethodDefinition:getPropertyName(){FunctionBody} MethodDefinition:setPropertyName(PropertySetParameterList){FunctionBody} GeneratorMethod:*PropertyName(UniqueFormalParameters){GeneratorBody} AsyncMethod:async[no LineTerminator here]PropertyName(UniqueFormalParameters){AsyncFunctionBody}
    1. Return HiddenMethodName of PropertyName.
    LiteralPropertyName:StringLiteral LiteralPropertyName:NumericLiteral ComputedPropertyName:[AssignmentExpression]
    1. Return empty.
    LiteralPropertyName:IdentifierName
    1. Return StringValue of IdentifierName.

    3.3HiddenNameList

    Editor's Note
    HiddenNameList of a PropertyDefinitionList or ClassElementList is a List containing all of the hidden names defined in an object literal or class body.
    PropertyDefinitionList:PropertyDefinition
    1. Return HiddenNameList of PropertyDefinition.
    PropertyDefinitionList:PropertyDefinitionList,PropertyDefinition
    1. Let names be HiddenNameList of PropertyDefinitionList.
    2. Append to names the elements of the HiddenNameList of PropertyDefinition.
    3. Return names.
    PropertyDefinition:IdentifierReference PropertyDefinition:PropertyName:AssignmentExpression PropertyDefinition:MethodDefinition
    1. Return a new empty List.
    PropertyDefinition:HiddenMethod
    1. Let name be HiddenMethodName of HiddenMethod.
    2. If name is empty, return a new empty List.
    3. Return a new List containing name.
    ClassElementList:ClassElement
    1. Return HiddenNameList of ClassElement.
    ClassElementList:ClassElementListClassElement
    1. Let names be HiddenNameList of ClassElementList.
    2. Append to names the elements of the HiddenNameList of ClassElement.
    3. Return names.
    ClassElement:MethodDefinition ClassElement:staticMethodDefinition ClassElement:;
    1. Return a new empty List.
    ClassElement:HiddenMethod ClassElement:staticHiddenMethod
    1. Let name be HiddenMethodName of HiddenMethod.
    2. If name is empty, return a new empty List.
    3. Return a new List containing name.
    InstanceVariableList:BindingIdentifier
    1. Return a new List containing StringValue of BindingIdentifier.
    InstanceVariableList:InstanceVariableList,BindingIdentifier
    1. Let names be HiddenNameList of InstanceVariableList.
    2. Append StringValue of BindingIdentifier to names.
    3. Return names.

    3.4ClassInitializer

    Editor's Note
    ClassInitializer of ClassElementList returns the first ClassInitializerBlock within a class definition. Early errors prevent multiple occurrances of ClassInitializerBlock within a single class definition.
    ClassElementList:ClassElement
    1. Return ClassInitializer of ClassElement.
    ClassElementList:ClassElementListClassElement
    1. Let head be ClassInitializer of ClassElementList.
    2. If head is not empty, return head.
    3. If ClassElement is ClassInitializerBlock, return ClassInitializerBlock.
    4. Return empty.

    3.5MultipleClassInitializers

    Editor's Note
    Early errors prevent multiple occurrances of ClassInitializerBlock within a single class definition.
    ClassElementList:ClassElement
    1. Return false.
    ClassElementList:ClassElementListClassElement
    1. If ClassInitializer of ClassElementList is true and ClassInitializer of ClassElement is true, return true.
    2. Return false.
